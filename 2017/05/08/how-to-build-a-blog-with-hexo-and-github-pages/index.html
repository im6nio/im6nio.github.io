如何使用 Hexo 和 GitHub Pages 搭建一個靜態博客

How to build a blog with Hexo and GitHub Pages

Posted by im6nio on 2017-05-08
#Hexo #Node.js #Git #GitHub #GitHub Pages

環境配置

安裝 Node.js

下載 : https://nodejs.org/en/download/

安裝 Git

下載 : https://git-scm.com/download/

安裝 Hexo

$ cd d:/hexo
$ npm install hexo-cli -g
$ cd blog
$ hexo init
$ npm install
$ hexo generate # 或者簡寫 hexo g
$ hexo server # 或者簡寫 hexo s，可以在 http://localhost:4000/ 預覽

Hexo 常用的指令

hexo generate (簡寫 hexo g) : 生成靜態檔案，會在當前目錄生成一個 public 資料夾。

$ hexo generate

hexo server (簡寫 hexo s) : 啟動本機 Web 服務，開啟瀏覽器輸入 http://localhost:4000/ 預覽，Ctrl + C 關閉本機 Web 服務。

$ hexo server
# INFO  Start processing
# INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

hexo deploy (簡寫 hexo d) : 部署到遠端平台。 (例如: GitHub)

$ hexo deploy

hexo clean : 清除快取檔案 db.json 和已產生的靜態檔案 public。

$ hexo clean

hexo new [layout] “文章名稱” (簡寫 hexo n [layout] “文章名稱”) : 新建一個文章，您可以在指令中指定文章的佈局（layout），預設為 post ，您可以透過修改 _config.yml 中的 default_layout 設定來指定預設佈局。

Hexo 有三種預設佈局 : post、page 和 draft，分別對應不同的路徑。

$ hexo new [layout] "文章名稱"

常用的組合指令

hexo s -g : 開啟預覽前生成靜態檔案。

$ hexo s -g

hexo d -g : 部署到遠端前生成靜態檔案。

$ hexo d -g

hexo g -d : 生成靜態檔案後部署到遠端。

$ hexo g -d

Hexo 主題設置

安裝主題

我以我使用的 Clean Blog 主題為例子 :

$ hexo clean
$ git clone https://github.com/klugjo/hexo-theme-clean-blog.git themes/clean-blog

啟用主題

修改 Hexo 目錄下的 _config.yml 文件中的 theme 屬性，將此屬性改為 clean-blog。

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: clean-blog

GitHub Pages 設置

創建自己的 GitHub Pages

  1. 創建 GitHub Pages 前，你需要先註冊一個 GitHub 帳號

  2. 在你的 GitHub 創建一個儲存庫 (repository)，儲存庫名稱一定要是 username.github.io，username 為你的 GitHub 帳號名稱

將 Hexo 部署到 GitHub Pages

使用 hexo deploy 部署

在使用 hexo deploy 部署前，需要先使用 npm 安裝 hexo-deployer-git 套件

$ npm install hexo-deployer-git --save

hexo deploy 可以部署到很多遠端平台，如果要部署到 GitHub 需要先到 _config.yml 中修改 deploy 屬性

deploy:
  type: git
  repo: git@github.com:username/username.github.io.git
  branch: master

以及確認是否有設定 SSH key ? 參考 GitHub Help

$ ssh-keygen -t rsa -b 4096 -C "user@email.com"

以上都設定完成，即可使用 hexo deploy 部署到遠端 GitHub

$ hexo deploy
